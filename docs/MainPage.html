La Suite NemesisRD 1.x es un conjunto de bibliotecas de clases escritas íntegramente en C++ cuyo objetivo es facilitar el desarrollo de aplicaciones profesionales.

Las facilidades que ofrece la Suite NemesisRD 1.x está especialmente orientado hacia el desarrollo de aplicaciones distribuidas, aunque también ha sido usada con éxito en el desarrollo de aplicaciones de consola.

Las plataformas nativas de NemesisRD 1.x son Linux y Solaris, tanto en 32 bits como en 64 bits.

Las facilidades más destacables podrían ser:

  • Desarrollo general de aplicaciones. Implementación optimizada de los patrones de diseño más populares, configuración de la línea de comandos, control de componentes, respuesta a señales de SO, interpretación y creación de documentos XML, sistema de trazas, capacidades multithreading, etc.
  • Uso en aplicaciones distribuidas. Sistemas de alta disponibilidad, tolerancia a fallos, reparto automático de carga, reparto automático de atención y control de la congestión. Permite la interoperabilidad con software de terceros al ofrecer módulos para tratar los protocolos más usados por la industria como: SSL, HTTP, HTTPS, SOAP, LDAP y SIP.
  • Conectividad a base de datos. Sistema de acceso a base de datos similar a JDBC que permite aislar el código de la aplicación del motor particular que se esté usando. Una aplicación podría estar conectada simultáneamente con un número interminado de servidores de base de datos; en el caso más extremo cada uno de estos servidores podría estar usando un motor (MySQL, Postgres u Oracle) distinto.
  • Tratamiento de objetos persistentes. Permiten instanciar las entidades persistentes guardadas en la base de datos como si fuera un objeto C++. El programador no tiene que preocuparse de cómo se transfiere la información del medio físico al objeto C++. Adicionalmente ofrece un sistema de acceso rápido, que optimiza el acceso a estos objetos C++.

    • Suite NemesisRD 1.x is a set of libraries and clases written in C++ whose main target is to make the professional application development easier.

    Features offered by the Suite NemesisRD 1.x are specially focussed on distributed application development, although it had been succesfully used to develop console application.

    The native platform of Suite NemesisRD 1.x are Linux 2.x and Solaris 5.x, both 32 bits and 64 bits.

    The main features are:

  • General development application. Improved implementation of most popular pattern designs, command line configuration,control of component application, OS signal processing, interpretation and creation of XML documents, logging system, multithreading capabilities, and so on.
  • Distribuited applications. Classes that offer high availability systems, fault tolerance, download and upload balancing, attencion balancing and control congestion. It enables interoperability with third parties software, offering a lot of popular industry protocol like: SSL, HTTP, HTTPS,SOAP, LDAP and SIP.
  • Database connecting. One database access system similar to JDBC that isolates the application code from the engine database in use. One application would be simultaneously connected to a unknown number of database servers, on the most extrem example, every database server could be using a different engine (mysql, postgre, oracle).
  • Persistent objects. Instantiate persistent entities previously recorded on the database like a common C++ instance. The programmer has not to think about how the information was transfered from the physical media to the C++ instance. As well it offers a quick access system, that optimize the access to the C++ persistent instances.


    • Librerías de la Suite NemesisRD
    nemesis
    		 Proporciona las clases y patrones de desarrollo basicos. Pueden ser usados en cualquier aplicacion y/o plataforma.
    nemesis.xml Proporciona las clases necesarias para la interpretación de documentos XML.
    nemesis.html Proporciona las clases necesarias para la interpretación de documentos HTML.
    nemesis.app Proporciona las clases necesarias para escribir aplicaciones sobre NemesisRD.
    nemesis.io Proporciona las clases necesarias para facilitar ciertas operaciones de entrada/salida.
    nemesis.comm Proporciona las clases necesarias para la comunicación entre procesos.
    nemesis.commsec Proporciona las clases necesarias para establecer comunicaciones seguras usando OpenSSL. Para mas informacion ver www.openssl.org
    nemesis.timex Proporciona las clases necesarias para el control de tiempos.
    nemesis.http Proporciona las clases necesarias para la comunicación entre procesos. Ver el documento de definicion de HTTP/1.1
    nemesis.ldap Proporciona las clases necesarias para el desarrollo de clientes LDAP.
    nemesis.dbms Data Base Management System (DBMS). Proporciona las clases necesarias para el acceso a base de datos.
    nemesis.dbms.mysql Proporciona las clases necesarias para el acceso a tablas de MySQL. Para mas información ver Documentacion del API C de MySQL
    nemesis.dbms.oracle Proporciona las clases necesarias para el acceso a tablas de Oracle (tm).
    nemesis.dbms.postgre Proporciona las clases necesarias para el acceso a tablas de PostgreSQL. Para mas información ver Documentación del API C de PostgreSQL
    nemesis.dbos Data Base Object System (DBOS). Proporciona las clases y templates que facilitan el tramiento de la informacion de la capa DBMS en forma de objectos C++
    nemesis.shmem Proporciona las clases necesarias para la gestión de memoria compartida entre procesos (Experimental). Ver la clase nemesis::shmem::StorageArea
    nemesis.tools Ofrece clases de alto nivel para las tareas usuales en las aplicaciones que desarrollamos normalmente.
    nemesis.sdp Proporciona las clases necesarias para la comunicación con procesos de la plataforma SDP original.
    nemesis.ims.sip Proporciona los componentes necesarios para la comunicacion entre procesos usando como capa de transporte Session Initiation Protocol (SIP) 2.0 definido en el RFC 3261. Para mas informacion RFC 3261.
    nemesis.ims.useragent Proporciona los componentes necesarios para el desarrollo de aplicaciones sobre IMS/SIP.
    nemesis.collaboration.sim Proporciona las clases necesarias para implementar facilmente un simulador especificado en base a un documento XML.
